    <article class="markdown-body entry-content p-3 p-md-6" itemprop="text"><h1><a id="user-content-py_ntnda_viewer-20200302" class="anchor" aria-hidden="true" href="#py_ntnda_viewer-20200302"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>PY_NTNDA_Viewer 2020.03.02</h1>
<p><strong>NOTICE: This code has been moved to areaDetector/ADViewers/Python/PY_NTNDA_Viewer</strong></p>
<p>Soon this repository will be deleted.</p>
<p>PY_NTNDA_Viewer is Python code that is similar to the Java EPICS_NTNDA_Viewer that comes with areaDector.</p>
<p>It is available in <strong>areaDetector/ADViewers/PY_NTNDA_Viewer</strong></p>
<p>It is a viewer for images obtained from an areaDetector channel that provides an NTNDArray.</p>
<p>There are currently 2 versions:</p>
<ol>
<li>P4P_NTNDA_Viewer.py
This uses <strong>p4p</strong>.</li>
<li>PVAPY_NTNDA_Viewer.py
This uses <strong>pvapy</strong>.
<strong>pvapy</strong> is not available on windows and does not provide connect/disconnect notification.</li>
</ol>
<p>Below there are instructions for</p>
<ol>
<li>Starting the example</li>
<li>Installation of required Python modules.</li>
</ol>

<h2><a id="user-content-theory-of-operation" class="anchor" aria-hidden="true" href="#theory-of-operation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Theory of Operation</h2>
<p>PY_NTNDA_Viewer provides support for an NTNDArray supported by <strong>areaDetector/ADCore/ADApp/ntndArrayConverterSrc</strong>.</p>
<p>PY_NTNDA_Viewer has the following python modules:</p>
<ol>
<li>P4P_NTNDA_Viewer.py : a channel provider that uses p4p to connect to an NTNDArray</li>
<li>PVAPY_NTNDA_Viewer.py : a channel provider that uses pvapy to connect to an NTNDArray</li>
<li>NTNDA_Viewer.py : The code that displays images provides by a channel provider.</li>
</ol>
<p><strong>NTNDA_Viewer.py</strong> defines the following python classes:</p>
<ul>
<li>NTNDA_Channel_Provider : a base class that a channel provider must implement</li>
<li>Image_Display : a class that displays an image</li>
<li>ImageControl: a class that calls Image_Display and provides sliders and zoom.</li>
<li>NTNDA_Viewer : A class that receives images from a channel provider.
It supports decompression and conversion of 1d numpy arrays to either 2d or 3d numpy arrays.
It then calls ImageControl with the 2d or 3d numpy array.</li>
</ul>
<h3><a id="user-content-an-ntdaarray-has-the-following-structure" class="anchor" aria-hidden="true" href="#an-ntdaarray-has-the-following-structure"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>An NTDAArray has the following structure:</h3>
<pre><code>epics:nt/NTNDArray:1.0
    union value                   // provider must present this a a 1d numpy array of one of the following types
        boolean[] booleanValue    // not used by PY_NTNDA_Viewer
        byte[] byteValue
        short[] shortValue
        int[] intValue
        long[] longValue
        ubyte[] ubyteValue
        ushort[] ushortValue
        uint[] uintValue
        ulong[] ulongValue
        float[] floatValue
        double[] doubleValue
    codec_t codec
        string name
        any parameters
    long compressedSize
    long uncompressedSize
    dimension_t[] dimension
        dimension_t
            int size
            // other fields not used by PY_NTNDA_Viewer
    // other fields not used by PY_NTNDA_Viewer
</code></pre>
<h3><a id="user-content-value" class="anchor" aria-hidden="true" href="#value"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>value</h3>
<p>The channel provider, e.g. <strong>PVAPYProvider</strong> or <strong>P4PProvider</strong>, must provide this as a 1d numpy array.
The mapping between the numpy dtype and the value type is:</p>
<pre><code>value type     dtype
----------     -----
byte           int8
short          int16
int            int32
long           int64
ubyte          uint8
ushort         uint16
uint           uint32
ulong          uint64
float          float32
double         float64
</code></pre>
<h3><a id="user-content-codec-1" class="anchor" aria-hidden="true" href="#codec-1"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>codec</h3>
<p><strong>codec</strong> is always of the form</p>
<pre><code>codec_t codec
    string name jpeg   // must be one  of "", jpeg, blosc, lz4, or bslz4
    any parameters
        int  1         // indicates dtype for each pixel
</code></pre>
<p>If <strong>codec.name</strong> is an empty string then <strong>value</strong> is not compressed.
In this case the rest of this section does not apply.</p>
<p>The code in <strong>NTNDA_Viewer.decompress</strong> decompresses the data in <strong>value</strong> and creates a 1d numpy array.
It first checks that the name is jpeg, blosc, lz4, or bslz4.
It is is not it generates an exception that results in a error message.</p>
<p>If the name is one of the supported types, the decompression code uses the following:</p>
<ul>
<li>The compressedSize and uncompressedSize from the NTNDArray passed by the channel provider</li>
<li>One of the shared libraries from: <strong>blosc</strong>, <strong>decompressJPEG</strong>,or <strong>bitshuffle</strong>.</li>
</ul>
<p><strong>codec.parameter.int</strong> is an integer representing the data types for the value
Note that for jpeg only dtype byte and ubyte are supported.
For the other codec types all dtypes are supported.</p>
<p>The mapping from the integer value to the numpy dtype is:</p>
<pre><code>    if typevalue== 1 : dtype = "int8"
    elif typevalue== 5 : dtype = "uint"
    elif typevalue== 2 : dtype = "int16"
    elif typevalue== 6 : dtype = "uint16"
    elif typevalue== 3 : dtype = "int32"
    elif typevalue== 7 : dtype = "uint32"
    elif typevalue== 4 : dtype = "int64"
    elif typevalue== 8 : dtype = "uint64"
    elif typevalue== 9 : dtype = "float32"
    elif typevalue== 10 : dtype = "float64"
    else : raise Exception('decompress mapIntToType failed')
</code></pre>
<p>The final result is the compressed data in <strong>value</strong> converted to an uncompressed 1d numpy array with the correct dtype.</p>
<h3><a id="user-content-dimension" class="anchor" aria-hidden="true" href="#dimension"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>dimension</h3>
<p>This is used by <strong>NTNDA_Viewer.dataToImage</strong> to transform the 1d numpy array to either a 2d or 3d numpy array.
If the number of dimensions is 2 than the data is for a monocromatic image.
If the dimension is 3 then one of the dimensions must have size 3 and must be RGB data.
Note also that areaDetector and numpy use different conventions for <strong>x</strong> and <strong>y</strong>.
Thus <strong>NTNDA_Viewer.dataToImage</strong> transposes x and y.</p>
